## The problem

My mornings used to start with scattered inputs: email, calendar changes, DMs, open tasks, and half-written notes from the night before. None of those tools were hard to open. The hard part was deciding what mattered before the day became reactive.

## The three-agent split

- Inbox reader: summarizes overnight messages and flags anything that needs a same-day answer.
- Reply drafter: writes first-pass responses in my voice but never sends them.
- Calendar guard: checks whether the morning still contains one protected deep-work block.

The important design choice was giving each agent one job. A single "morning assistant" sounded clean but became vague quickly.

## What changed

The system does not make decisions for me. It reduces the surface area of the decision. I open one page and see what changed, what needs a reply, and what should wait.

## The rule that made it work

Every output must be editable and disposable. If an agent writes something awkward, I delete it. If it summarizes badly, I correct the prompt. The workflow improves because the failure is visible.
